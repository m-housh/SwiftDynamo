DynamoQuery

public struct DynamoQuery

Holds database input and options for a database operation. You typically do not work directly with a query, but with a DynamoQueryBuilder instead.

  • action

    The action we are taking on the database.

    Declaration

    Swift

    public var action: Action
  • schema

    The schema we are interacting with. This at least needs to have the table-name, but can optionally hold a partition key and / or a sort-key.

    Declaration

    Swift

    public var schema: DynamoSchema
  • input

    A stack for values to be saved to the database. This should in most cases not be manipulated directly as placing the wrong values could cause fatal errors to occur when running the query.

    Declaration

    Swift

    public var input: [Value]
  • sortKey

    An optional convenience to set / override a sort key on the query.

    This can be set from several different places, globally on the schema, declared as a property on the model, or set directly on the query builder. That means that whatever gets set last is what gets set on the request.

    Declaration

    Swift

    public var sortKey: (String, Value, Filter.Method)? { get set }
  • partitionKey

    An optional convenience to set / override a partition key on the query.

    This can be set from several different places, globally on the schema, declared as a property on the model, or set directly on the query builder. That means that whatever gets set last is what gets set on the request.

  • options

    A stack of options set on the query. Most of these are aws options that we allow a user to pass in if needed, the ones that we directly use will have an abstraction around them, however they get stored in order in the stack, so if the some option re-occurs whatever was set last will win.
